# Project 2 — CI/CD Pipeline with GitHub Actions + S3

## What I Built
An automated CI/CD pipeline that deploys a static 
website to AWS S3 automatically whenever code is 
pushed to GitHub.

## How it Works
Developer pushes code to GitHub
→ GitHub Actions workflow triggers automatically
→ Website files deploy to AWS S3
→ Website is live without manual work!

## AWS Services Used
- Amazon S3 (Static Website Hosting)
- IAM (GitHub Actions permissions)

## DevOps Tools Used
- GitHub Actions
- YAML workflow file

## What I Did
- Created S3 bucket with static website hosting
- Created IAM user with S3 permissions for GitHub Actions
- Wrote GitHub Actions workflow file (main.yml)
- Configured GitHub Secrets for AWS credentials
- Tested automatic deployment by pushing code

## What I Learned
- How CI/CD pipelines work
- How GitHub Actions automates deployments
- How IAM permissions work with GitHub Actions
- How to write a YAML workflow file
